# MESS-Materials v0.1 — Producer Plan

_Status: draft, committed at repo bootstrap (v0.0 scaffolding)._

## Scope

v0.1 ships **Tier 1 DFT-computed properties** for MES materials across
all component classes — electrodes (anode, cathode), membranes (proton
exchange, anion exchange), chambers, current collectors, catalysts,
gaskets, and support structures — keyed by the MESS-Parameters
material-slug vocabulary. No composite scores, no Tier 2 data
(elasticity, surface energies), no MLIP features — those land in v0.2+.

Tier 1 = MP summary API (band_gap, is_metal, formation_energy,
energy_above_hull, density) + Pourbaix stability at 3 MES operating
points + crystal structure (CIF) for later MLIP use.

Split into two sub-tags:
- **`v0.1.0-pilot`** — 100 slugs, validates the pipeline end-to-end.
- **`v0.1.0`** — 500 slugs, primary coverage release.

## What this repo is

A sidecar to `Messai-io/MESS-Parameters`. MESS-Parameters owns the MES
parameter ontology and literature extractions. This repo owns DFT-computed
properties for each material-named parameter slug in that ontology. The
two communicate via shared slugs only; neither modifies the other's
schema.

Downstream consumer: `Messai-io/messai-ai`. It pins this repo as a
submodule once v0.1.0 is tagged, splits rich.json into per-slug JSON
fixtures, and renders the Computed Properties section on parameter detail
pages.

## Deliverables — v0.1.0-pilot (100 slugs)

### Scripts (Python, invoked via `make` or `pnpm`)
- `scripts/extract-mes-material-slugs.py`
  - Reads pinned MESS-Parameters submodule
    (`open-source/MESS-Parameters/data/parameter-definitions-rich.json`).
  - Filters entries whose category is material-related (ELECTRODE,
    CATHODE, ANODE, MEMBRANE, CURRENT_COLLECTOR, CATALYST, MATERIALS).
    Check actual enum values in `rich-parameters.schema.json`.
  - Emits `data/mess-material-slugs.json` — flat list of slugs + usage_count.
- `scripts/rank-slugs-by-usage.py`
  - Sorts by `usage_count` descending. Output: `data/slug-ranking.json`.
- `scripts/fetch-mp.py`
  - Uses `mp-api` client. Reads `data/slug-to-mp.yaml`, fetches summary
    fields + CIF structure per MP material_id. Caches raw responses under
    `data/mp-cache/` (check in — reproducibility).
- `scripts/compute-pourbaix.py`
  - Uses pymatgen's `PourbaixDiagram`. Evaluates each material at three
    conditions: `(-0.3 V, pH 7)` (MFC anode), `(+0.2 V, pH 7)` (MFC
    cathode), `(-0.7 V, pH 7)` (MEC cathode). Outputs per material:
    `{stable, passivated, corroding}` plus the decomposition product if
    unstable. Writes `data/pourbaix-results.json`.
- `scripts/assemble-rich-json.py`
  - Joins MP summary, structure, and Pourbaix results into one
    `data/mp-materials-rich.json` keyed by MESS-Parameters slug.
    Composites aggregate component data under a `components: [...]` array.

### Data artifacts
- `data/slug-to-mp.yaml` — **manual curation file** (the single
  high-effort input). 100 entries in v0.1.0-pilot, 500 in v0.1.0. Shape:
  ```yaml
  - slug: carbon_cloth
    components:
      - mp_id: mp-48           # graphite
        role: bulk_structure
        proxy: true             # flag: amorphous material, proxied to crystalline
        confidence: low
  - slug: mno2_on_carbon
    components:
      - mp_id: mp-510408
        role: catalyst
      - mp_id: mp-48
        role: support
  - slug: platinum_cathode
    components:
      - mp_id: mp-126
        role: bulk_structure
  ```
- `data/mp-materials-rich.json` — assembled output; one record per MESS
  slug; schema-validated against `schemas/mp-material.schema.json`.
- `data/mess-parameters-lock.json` —
  `{mess_parameters_tag: "v0.2.0", mapped_slugs: N, unmapped_slugs: M,
  generated_at: ISO8601}`.
- `data/unmapped-materials.json` — every MESS slug not in `slug-to-mp.yaml`,
  with a required `reason` field. Examples: `amorphous, no crystalline
  analog`, `proprietary composition`, `not yet curated`.
- `data/mp-cache/<mp_id>.json` — raw MP API responses, checked in.
- `data/pourbaix-results.json` — raw Pourbaix output for audit.

### Schemas
- `schemas/mp-material.schema.json` — JSON Schema for rich.json records.
  Mirrors MESS-Parameters' pattern
  (`open-source/MESS-Parameters/schemas/parameter.schema.json`). Key fields:
  `slug` (string, required), `components` (array), `band_gap_eV`,
  `is_metal`, `formation_energy_eV_per_atom`, `energy_above_hull_eV_per_atom`,
  `density_g_per_cm3`, `pourbaix: {mfc_anode, mfc_cathode, mec_cathode}`,
  `mp_snapshot_date`, `mp_functional` (enum: PBE | r2SCAN),
  `confidence_tier` (enum: high | medium | low), `cif` (string, the
  structure as text).

### Documentation
- `README.md` — purpose, relationship to MESS-Parameters, how to
  regenerate, how to consume.
- `data/SCIENTIFIC_INTEGRITY.md` — the caveats (amorphous-carbon proxy
  limitation; Pourbaix = thermodynamics only, will false-negative
  passivating alloys; r2SCAN vs PBE; MP CC-BY-4.0 attribution
  requirement; confidence-tier semantics).
- `docs/consumer-contract.md` — the shape messai-ai can depend on.
  Producer-side counterpart to the consumer's
  `docs/consuming-mess-parameters.md`.

### CI
- `ci/validate.yml` — runs `ajv validate` against every rich.json record
  on every push. Additionally checks that every MESS-Parameters material
  slug appears in exactly one of `slug-to-mp.yaml` or
  `unmapped-materials.json`; fails the build on drift.

### Pipeline validation checklist — must pass before tagging v0.1.0-pilot
(a) Fresh clone + `pip install -r requirements.txt` + `make build-data`
    reproduces all `data/*.json` artifacts deterministically.
(b) Spot checks pass for ≥5 anchor materials — graphite `mp-48`
    band_gap must match MP web UI; platinum `mp-126` must show
    `is_metal: true`; rutile TiO₂ must have non-zero band_gap; etc.
(c) Pourbaix sanity anchors pass: Pt at (+0.2V, pH 7) = stable; Fe at
    (-0.3V, pH 7) = corroding; Ti at any MES condition = passivated
    (stable via TiO₂ layer — confirms the passivator handling).
(d) `ajv validate` green on every record.
(e) `unmapped-materials.json` + `slug-to-mp.yaml` together cover every
    material-category slug in MESS-Parameters v0.2.0.

## Scaling from pilot to v0.1.0 (500 slugs)

Only after v0.1.0-pilot is tagged:
- Expand `slug-to-mp.yaml` to 500 entries. Expect a higher unmapped
  rate at the tail.
- Batch-run within the MP API free-tier rate limit (~1000 req/day). Add
  a `--resume` flag to `fetch-mp.py` so a run can be interrupted and
  continued.
- Confidence-tier assignment: `high` for direct crystalline MP match;
  `medium` for composites; `low` for amorphous proxies (all carbon
  fabrics → `mp-48`).
- Regression check: rerun the v0.1.0-pilot anchor checks against the
  500-slug dataset. Byte-for-byte match, or every diff documented.

- **EquiformerV2, OMat24:** Meta FAIR — review licenses when those
  phases activate. Not required for v0.1.

## Coordination with upstream

- MESS-Parameters is pinned at `v0.2.0` for v0.1.0-pilot.
- If MESS-Parameters tags a new version that adds or renames material
  slugs, MESS-Materials' next release must either map the new slugs or
  list them in `unmapped-materials.json`. CI enforces this.
- Submodule drift between what this repo pins and what messai-ai pins
  is flagged by messai-ai's `pnpm pre-deploy` (producer-side action:
  bump in coordination, not in isolation).

## Out of scope for v0.1

- Composite scores (EVS, BCS, LSS) — v0.2.
- Elasticity, surface energies, dielectric, phonon — v0.2 or later.
- MACE/MLIP structural embeddings — v0.3, sourced from MESS-Learning.
- WBM novel-materials screening — v0.4.
- MatBench-MES benchmark curation — long-term, paired with
  MESS-Learning.
